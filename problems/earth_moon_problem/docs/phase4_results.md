## Per-row extraction recipe

Given a run directory `data/<label>/<ts>/` produced by `run_smsg.sh`:

- **weights, n, m, slack** — from `meta.txt` (the `# weights=...` line our
  `earthmoon_blowup.py` writes to stderr) cross-referenced with
  [`cycle_blowup.py verify`](../scripts/cycle_blowup.py).
- **SMS result** — `SAT` if `stdout.log` contains a `Solution N` line;
  `UNSAT` if it contains `Search finished` and zero `Solution` lines and
  `exit_status == 0`; `TIMEOUT` if `exit_status == 124`.
- **wall time** — `end_utc - start_utc` from `meta.txt` (or `5062` for the
  candidate1 calibration). This is the wrapper's wall-clock, including
  Python startup and DIMACS generation.
- **total time** — grep `Total time:` in `stdout.log`. SMS-internal only,
  excludes Python startup / encoding emission.
- **propagator calls** — grep `Calls propagator:` in `stdout.log`.
- **Kuratowski clauses added** — the `Added clauses:` field under
  `Statistics for ThicknessTwoChecker:` in `stdout.log`. (Distinguishes
  planarity work from symmetry-breaking; MinimalityChecker has its own
  separate `Added clauses` counter, not what this column reports.)

## Conventions

- A SAT result is **suspect** unless verified by an independent biplanarity
  test. Replay the certificate through `networkx.check_planarity` on each
  layer, with the layer assignment derived from the directed-graph
  encoding per KSS §5.2.
- An UNSAT row earns a brief follow-up commit: a `data/<label>/README.md`
  describing the run and any anomalies, plus a row update here.
- A TIMEOUT row is informative even though indeterminate; record peak
  in-flight stats from the truncated `stdout.log`.
